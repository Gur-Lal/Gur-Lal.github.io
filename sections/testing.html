5. Usability Testing

Testing


After completing my high-fidelity prototype, I conducted usability testing with 10+ participants. I asked each person to complete a set of realistic scenarios, including:

  • Add a new medication
  • Edit an existing medication
  • Log a health entry
  • Navigate back to the dashboard

I used a think-aloud method so I could understand not just what users were doing, but what they were expecting.


Key Findings


Several participants struggled to move between sections. When they finished adding a medication, they weren’t sure how to return to the main dashboard. Some relied on the browser back button instead of the interface itself. This showed me the navigation wasn’t clear or consistent enough. When asked to edit a medication, multiple users tapped the medication name expecting it to open details. The edit action wasn’t immediately obvious to them. They hesitated before finding the correct interaction. During the health logging task, a few participants paused because they weren’t sure if the fields were editable or informational. The visual hierarchy made some inputs feel static.


Improvements Made


Based on testing, I made the following changes in the final design:

  • Added a clearer and more consistent navigation structure so users can easily move between Home, Medications, and Health Tracking without relying on browser controls.
  • Made medication cards fully clickable to open details, aligning with what users naturally tried to do during testing.
  • Strengthened visual cues on editable fields by refining borders, spacing, and input styling so they clearly look interactive.
  • Improved screen hierarchy to make primary actions more visually dominant.

After these updates, task completion became faster and users expressed more confidence navigating the app.
